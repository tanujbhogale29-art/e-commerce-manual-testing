# 📋 Test Plan — E-Commerce Website

**Project Name:** E-Commerce Website Manual Testing  
**Prepared By:** Tanuj Bhogale  
**Date:** March 2026  
**Version:** 1.0

---

## 1. Objective

To verify that the core modules of the E-Commerce website — Login, Search, Cart, and Checkout — function correctly as per requirements, and to identify and report defects.

---

## 2. Scope

### In Scope
- Login Module
- Search Module
- Cart Module
- Checkout Module

### Out of Scope
- Payment Gateway Integration (live)
- Admin Panel
- Performance Testing
- Mobile Responsiveness

---

## 3. Test Strategy

| Type | Description |
|------|-------------|
| Functional Testing | Verify each feature works as expected |
| Boundary Value Analysis | Test at min/max input limits |
| Equivalence Partitioning | Divide inputs into valid/invalid classes |
| Regression Testing | Re-test after bug fixes |

---

## 4. Test Environment

| Item | Details |
|------|---------|
| OS | Windows 10 |
| Browser | Chrome 120, Firefox 121 |
| Application | E-Commerce Website (Staging) |
| Test Data | Dummy user accounts, products |

---

## 5. Entry Criteria

- Application is deployed and accessible
- Requirements are reviewed
- Test cases are written and reviewed

---

## 6. Exit Criteria

- All 15 test cases executed
- All critical/high bugs fixed and verified
- Test Summary Report prepared
